En général, les vidéos sont téléchargées sur {{site.product_short}} à l'aide du module Upload. Ces fichiers sont ensuite gérés par Brightcove. Avec les ressources distantes, vous conservez vos fichiers vidéo sur votre propre système ou sur un système tiers, puis vous fournissez à {{site.product_short}} l'URL du fichier vidéo.

Les vidéos de ressources distantes peuvent également être créées à l'aide de l'API CMS. Pour plus d'informations, consultez la documentation destinée aux développeurs. Les vidéos de ressources distantes créées à l'aide de l'API CMS peuvent ensuite être gérées à l'aide du module Media.

Accès aux ressources distantes

Tous vos fichiers de ressources distants doivent être disponibles sur une URL publique sans aucune exigence d'authentification ou de contrôle d'accès. Si vos ressources distantes sont hébergées par un CDN, l'URL des ressources distantes doit être l'URL des ressources du CDN et non l'URL du serveur d'origine.

Vidéo sur la création d'un actif distant

Pour créer une vidéo sur les ressources distantes, procédez comme suit.

  1. Ouvrez le module Médias.
  2. Cliquez sur Ajouter une vidéo distante dans le panneau de navigation latéral. La boîte de dialogue Ajouter une vidéo distante s'ouvre.
  3. Saisissez le nom de la vidéo. Le nom de la vidéo sera affiché dans le module Médias et également dans le lecteur.
  4. Sélectionnez le type de vidéo, Live ou On Demand.
    5. Veuillez noter que Live prend uniquement en charge HLS et On Demand prend en charge HLS, MP4 et DASH.
  5. Pour les types de vidéos à la demande, saisissez la durée au format hh:mm:ss.
  6. Pour chaque format associé, entrez l'URL et le format du format associé. Cliquez sur Ajouter un format associé.
    7. Remarque: Les URL relatives aux protocoles sont prises en charge.
  7. Cliquez sur Enregistrer pour enregistrer la ressource distante. La vidéo de la ressource distante sera répertoriée dans le module Médias après l'actualisation de la page.

Les vidéos de ressources distantes ne comporteront pas d'affiches ou de vignettes, mais les images peuvent être téléchargées. Pour plus d'informations sur le téléchargement d'affiches et de vignettes, consultez la section Chargement d'images d'affiches et de vignettes.

Limitations

  • Vous ne pouvez pas utiliser la fonction de capture d'image pour créer des images fixes et des images miniatures pour les vidéos de ressources distantes. Toutefois, les images fixes et miniatures vidéo peuvent être téléchargées pour les vidéos de ressources distantes
  • Les vidéos de ressources distantes créées dans des comptes Basic ne peuvent pas être modifiées après leur création
  • SSAI n'est pas pris en charge pour les ressources distantes
  • Les DRM ne sont pris en charge que si vous pouvez personnaliser le lecteur pour communiquer avec le serveur de licences DRM. C'est possible avec le lecteur Brightcove via un plugin. Pour les autres lecteur s, vous devrez consulter la documentation du lecteur.
  • Analytics: le système Brightcove Analytics dépend des métadonnées vidéo générées par le système, telles que la durée de la vidéo. Les données d'analyse des ressources distantes seront moins riches si vous ne pouvez pas fournir ces métadonnées pour la ressource distante.
  • Garantie de lecteur: toutes les vidéos ingérées dans {{site.product_short}} peuvent être lues dans le lecteur Brightcove. Cela est vrai pour presque tous les actifs distants, mais dans certains cas extrêmes, les vidéos peuvent ne pas être lues. Il existe quelques fonctionnalités avancées de DASH et HLS que nous ne prenons pas encore en charge, par exemple. Pour les lecteurs autres que Brightcove, vous devrez vérifier que le format vidéo est entièrement pris en charge par ce lecteur.
  • Les spectateurs peuvent ne pas être en mesure de lire les vidéos des ressources distantes si l'heure de leur ordinateur est réglée en avance sur l'heure réelle
  • Les points de repère ne peuvent pas être ajoutés aux vidéos de ressources distantes en
  • Il n'est pas possible de convertir un actif VOD normal existant en un actif distant
