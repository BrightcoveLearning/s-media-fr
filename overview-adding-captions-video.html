Présentation

Des sous-titres codés peuvent être ajoutés aux vidéos pour fournir des informations supplémentaires ou interprétatives aux personnes qui souhaitent y accéder. Les sous-titres sont nécessaires pour que les personnes sourdes et malentendantes puissent accéder à la partie audio des présentations vidéo.

Ce texte peut être utilisé à des fins d'accessibilité, pour fournir des traductions de la piste audio ou à d'autres fins, mais la technologie sous-jacente utilisée est la même.

Emplacement du fichier de sous-titres

Vous pouvez choisir où héberger le fichier de sous-titres WebVTT:

  1. À une URL distante (ressource distante): dans ce cas, le fichier WebVTT est hébergé sur votre serveur et vous ajoutez l'URL du fichier aux métadonnées de la vidéo.
    Remarque: Si vous hébergez des fichiers de sous-titres sur votre propre serveur, assurez-vous que le fichier se trouve sur une URL accessible par le lecteur Brightcove. Le serveur hébergeant le fichier de sous-titres doit également avoir activé le partage de ressources inter-origines (CORS). Pour en savoir plus sur CORS, cliquez ici.
  2. Téléchargé en utilisant {{site.product_short}} (ingested) - Dans ce cas, le fichier WebVTT est chargé (ingéré) sur {{site.product_short}}. Cela peut être fait à l'aide du module Media ou des API.

Remarque: Si le fichier source vidéo que vous intégrez dans {{site.product_short}} contient des légendes 608/708 intégrées, elles seront ignorées pendant le traitement de la vidéo et ne seront pas présentes dans les rendus vidéo. Pour ajouter des sous-titres à vos vidéos, plusieurs options s'offrent à vous:

  • Laissons {{site.product_short}} générer des fichiers de sous-titres
  • Ajout manuel de légendes
  • Utiliser l'API Dynamic Ingest
  • Utiliser l'API CMS (pour les sous-titres hébergés à distance)

Associer des sous-titres à une vidéo

Il existe plusieurs manières d'associer un fichier de sous-titres à une vidéo:

  1. Utilisation de Studio
  2. Modifier le code de publication du lecteur
  3. Utilisation des API

Génération automatique de sous-titres pour les vidéos

{{site.product}} peut générer des sous-titres pour les vidéos. Pour plus d'informations, consultez la section Génération de sous-titres pour les vidéos.

Ajout de légendes dans Studio

Les sous-titres peuvent être associés à une vidéo à l'aide du module Médias. Pour plus d'informations, consultez la section Ajout de sous-titres aux vidéos à l'aide du module Médias.

Modifier le code de publication du lecteur

Dans ce cas, une balise <track> est ajoutée au code de publication du lecteur. Pour plus d'informations, consultez la section Ajout de sous-titres aux vidéos par programmation.

Ajout de légendes via les API

La façon dont vous ajoutez des sous-titres via les API varie selon que le fichier de sous-titres sera ingéré ou utilisé en tant que ressource distante.

  • Légendes des ressources distantes - Si vous préférez héberger les sous-titres vous-même (ou si un fournisseur de sous-titres les héberge), vous pouvez associer les sous-titres à une vidéo à l'aide de l'API CMS. Pour plus d'informations, consultez la section Utilisation de l'API CMS: Ajoutez des légendes WebVTT.

  • Ajouter des légendes à {{site.product}} - Si vous souhaitez que Brightcove héberge vos sous-titres, vous pouvez intégrer votre fichier WebVTT à l'aide de l'API Dynamic Ingest. Pour plus d'informations, consultez la section Ingestion de fichiers WebVTT.

Remarque: Quelle que soit la méthode utilisée pour ajouter des fichiers de sous-titres à une vidéo, vous ne pouvez pas ajouter plus de 1 000 pistes de texte.

Suppression des légendes

Les sous-titres peuvent être supprimés des vidéos à l'aide du module Médias ou via l'API CMS. Dans Studio, ouvrez le module Media et modifiez les propriétés de la vidéo. Modifiez la propriété Text Tracks si nécessaire.

Les sous-titres peuvent également être supprimés à l'aide de l'API CMS en effectuant une demande de mise à jour de la vidéo et en soumettant un corps de demande avec un tableau text_tracks vide (ou un tableau dont l'objet associé à cet ensemble de légendes a été supprimé).

Légendes de style

Le style peut être appliqué aux légendes WebVTT à l'aide de CSS. Voir Légendes de style pour plus de détails.
